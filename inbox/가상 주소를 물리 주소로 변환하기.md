**Virtual Address to Physical Address Conversion**

The virtual address to physical address conversion process involves the translation of virtual addresses into physical addresses using the Translation Lookaside Buffer (TLB) and page tables.

### Parameters

* Virtual Address: 14 bits
* Physical Address: 12 bits
* Page Size: 64 bytes, with VPO/PPO = 6 bits
* VPN: 8 bits (bits 13-6)
* TLB: 16 entries, 4-way associative, with TLBI = 2 bits and TLBT = VPN bits[7:2]
* Cache: 16 lines, direct-mapped, with 4-byte blocks, using CO = 2 bits, CI = 4 bits, and CT = 6 bits

### Address Translation Process

The address translation process consists of two stages: TLB lookup and page table lookup.

#### Virtual Address Examples

##### (1) Virtual Address: 0x027C
TLB Set 2 lookup with TLBT = 0x00 yields a **TLB miss**. Page table lookup with VPN = 0x02 yields Valid = 1 and PPN = 0x33, resulting in Physical Address: PA = PPN << 6 | VPO = 0x33 << 6 | 0x3C = 0xCFC.

##### (2) Virtual Address: 0x03A9
TLB Set 2 lookup with TLBT = 0x03 yields a **TLB miss**. Page table lookup with VPN = 0x0E yields Valid = 1 and PPN = 0x11, resulting in Physical Address: PA = 0x11 << 6 | VPO = 0x29 = 0x469.

### Cache Breakdown

The cache breakdown for both virtual addresses is as follows:

* CO = bits[1:0] = 00 (0x0) or 01 (0x1)
* CI = bits[5:2] = 1111 (0xF) or 1010 (0xA)
* CT = bits[11:6] = 110011 (0x33) or 010001 (0x11)
